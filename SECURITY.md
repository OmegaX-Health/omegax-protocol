# Security Policy

## Supported versions

Security fixes are prioritized for the latest public default branch and the latest tagged public release line.

## Reporting a vulnerability

Please do **not** open public issues for security reports.

Use one of these channels:

1. Open a private GitHub Security Advisory in this repository, or
2. Contact the maintainers through an established private security/support channel.

Please include:
- affected commit, tag, or deployment
- impact summary
- reproduction steps or proof of concept
- suggested remediation if available

## Disclosure expectations

- We will acknowledge valid reports as quickly as practical.
- We may coordinate an embargo window before public disclosure when users would be put at risk by immediate publication.
- Public writeups should wait until a fix or mitigation is available.

## Audit status

Unless explicitly stated in release notes or repository documentation, do not assume a formal third-party security audit has been completed for the current revision.
