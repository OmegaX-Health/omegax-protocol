# Genesis Protect Acute Actuarial Pricing Workbook

This folder contains the repo-first actuarial pricing and launch-control workbook for the first Genesis Protect Acute policies: Event 7 and Travel 30.

The review is intentionally a launch-gate model, not a formal external actuarial opinion. It tests whether the first policies can be issued with explicit reserve floors, SKU-specific pause controls, and country/cohort gating that match the current public metadata and claim simulation surface. The April 27 p99.5 pricing redesign is now approved and applied in repo metadata.

## Files

- `assumptions.json` records locked SKU inputs, reserve assumptions, country posture, and source references.
- `scenario-matrix.json` defines the deterministic launch, growth, adverse, correlated-event, LP-exit, and gating scenarios.
- `review-output.json` is generated by `npm run genesis:actuarial:review`.
- `review-memo.md` is the generated human-readable workbook summary.
- `canonical-update-plan.md` is the generated synchronization plan for repo metadata, docs, and Notion changes.

## Regenerate

```sh
npm run genesis:actuarial:review
```

The generator validates the assumptions against the current public Genesis metadata before writing output. If premiums, cover windows, caps, benefit style, or waiting periods drift, the review fails instead of silently producing stale gates.

## Current Launch-Gate Conclusions

- Treat Travel 30 as the primary live launch SKU and Event 7 as the fast-demo/event SKU.
- Use the approved p99.5 design: Event 7 stays `$39` retail with a `$1,000` cap; Travel 30 uses `$159` retail with a `$3,000` cap.
- Use approved cohort bands of `$28-$36` for Event 7 and `$112-$144` for Travel 30.
- Keep public-open issuance around `1,000 Event 7 + 500 Travel 30` at baseline until additional LP or sponsor backstop is posted.
- Keep Event 7 at a hard cap of `50` members per named venue/event in Phase 0.
- Treat Travel 30 cohorts above `100` concentrated or illness-linked members as requiring sponsor/backstop review before issuance.
- Use country posture as an operational gating layer: `available`, `sponsor_only`, `waitlist`, and `unavailable`.
- Pause only the affected SKU by default when reserve thresholds are breached, unless operators intentionally widen the pause.
- Keep public metadata, generated workbook artifacts, and durable Notion doctrine synchronized to the approved pricing redesign.

## Limitations

- The claim simulation file is used for scenario coverage evidence, not jurisdiction-level pricing credibility.
- The model uses deterministic stochastic severity assumptions, not paid-claim triangles.
- Correlated-event scenarios are deterministic stress checks, not stochastic catastrophe models.
- Country posture is an operational readiness taxonomy and does not replace legal, regulatory, sanctions, or payment-rail review.
